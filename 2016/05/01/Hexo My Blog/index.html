    什么是Hexo？

    Hexo 是一个快速、简洁且高效的博客框架。使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。据说作者是一位来自台湾的大学生tommy351，表示深深的膜拜。接下来就是用Hexo搭建博客的过程, hexo.io/zh-cn/ 这是Hexo的中文官网(注意可能有墙),上面有文档查看、主题下载。由于我用的是Mac,所以本文主要就是针对Mac端,用Hexo+github搭建个人博客啦啦啦啦(其实win端步骤也差不多的)。首先捋一下过程：

    1.首先hexo是基于node.js的，所以必须安装node.js

    2.安装node.js方法很多，我选择homebrew安装方式(简单粗暴)

    3.安装homebrew就很简单了，mac自带ruby脚本功能，一句话OK

    4.hexo提交部署github需要使用git工具，所以需要安装git，用homebrew的话也是一句话搞定

    5.整理一下安装顺序（homebrew-node.js-hexo-git）

    OK，思路清晰了，开工！

    下载Homebrew

    ruby -e "$(curl -fsSL https://raw.github.com/Homebrew/homebrew/go/install)”

    安装Node.js

    brew install node

    安装Hexo

    1.全局安装Hexo

    npm install -g hexo

    如果遇到错误，应该是被墙堵住了，那就要这样安装

    npm install -g hexo-cli --no-optional

    2.创建blog文件夹(需要进入到自己要存放的目录下再执行，不然默认是在Mac的个人这个文件夹下创建) 

    hexo init blog

    3.进入blog文件夹

    cd blog

    4.安装Hexo依赖包 

    npm install

    5.运行服务器（默认地址为 http://localhost:4000 在浏览器打开就可以预览到默认主题的博客首页啦）

    hexo server  （ 缩写是 hexo s ）

    注意：

    1.如果遇到因为权限问题报错，在命令最前面加 -sudo

    2.这里不需要运行 hexo g 指令，我们打开访问的页面是node.js动态生成的，也就是本地网页。

    安装Git

    在这里我试了2种方法，第一种是直接用homebrew下载，第二种是下载Xcode，因为Xcode自带git，但是下载Xcode，显得有些累赘，如果不是iOS开发，Xcode就没啥用还占空间，当然如果是IOS开发就可以用第二种方法,简单粗暴。但在这里我推荐第一种——直接用homebrew下载：

    sudo brew install git
    注册一个github账号以及创建一个仓库用来存放博客文件

    这个就没什么好说的啦，简单操作。(实在不会就看看这个链接http://www.jianshu.com/p/13e64c9e2295)

    配置Git

    1.检查SSH keys是否存在

    cd ~/.ssh

    如果显示“No such file or directory”，跳到第3步，否则第2步。

    2.备份和移除原来的SSh key设置(因为已经存在key文件，所以需要备份旧的数据并删除)

    在相应路径下（~/.ssh/id_rsa.pub）找到id_rsa和id_rsa.pub两个文件,剪切到别的地方备份好。

    3.生成新的SSH Key(注意将your_email@example.com换成自己注册Github的邮箱地址。后面只需要默认设置就好，所以当需要输入文件名的时候，果断回车。)

    $ ssh-keygen -t rsa -C "your_email@example.com"

    接着系统会让你输入密码

    Enter passphrase (empty for no passphrase):<输入密码>
Enter same passphrase again:<再次输入密码>

    然后就默认会在相应路径下（~/.ssh/id_rsa.pub）生成id_rsa和id_rsa.pub两个文件，找到id_rsa.pub文件，用文本编辑器打开，复制出来。
    登陆github，点击右上角的 Account—>Settings—>SSH Public keys —> add another public keys
    把刚刚复制的SSH Key到里面（key文本框中），点击 add key 就ok了。

    最后我们来测试一下SSH Key是不是配置好了

    ssh -T git@github.com

    如果是这样的：

    The authenticity of host 'github.com (207.97.227.239)' can't be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

    淡定，输入yes 就会看到

    Hi <em>username</em>! You've successfully authenticated, but GitHub does not provide shell access.

    到这里只是能通过SSH Key链接github，但是还没完，为了通过git来提交博客的文章，我们还需要配置好git的账号

    git config --global user.name "你的github名字"
git config --global user.email "你注册github的邮箱"

    到这里才算是真正配置好Git了，下面就开始配置自己的博客啦。

    配置博客

    注意：在配置所有的_config.yml文件时（包括theme中的），在所有的冒号:后边都要加一个空格，否则执行hexo命令会报错，切记 切记

    本地的blog文件夹下内容为：

    _config.yml    
db.json 
node_modules 
package.json
scaffolds
source
themes

    首先，把github部署为博客的仓库。

    1.打开blog文件夹下的_config.yml，往下滑到最后，修改成下面这样：

    deploy:
    type: git
    repository: https://github.com/Wenghaobin/wenghaobin.github.io.git
   branch: master

    2.在blog文件夹目录下执行生成静态页面命令：

    hexo generate        或者：hexo g

    此时若出现如下报错：

    ERROR Local hexo not found in ~/blog
ERROR Try runing: 'npm install hexo --save'

    则执行命令：

    npm install hexo --save

    若无报错，自行忽略此步骤。

    3.再执行配置命令：

    hexo deploy            或者：hexo d

    注意：若执行命令hexo deploy仍然报错：无法连接git，则执行如下命令来安装hexo-deployer-git，
    安装完应该出现在node_modules文件夹下

    npm install hexo-deployer-git --save

    然后再次执行hexo generate和hexo deploy命令，在浏览器中打开网址http://wenghaobin.github.io（将wenghaobin换成你的用户名）能看到http://localhost:4000时一样的页面。

    OK!博客和github连接成功！

    接着更换主题

    哈哈，作为爱装b的新时代骚年，一定要选一个自己喜欢的主题才行！

    你可以到Hexo官网主题页去搜寻自己喜欢的theme。这里以hexo-theme-indigo为例

    终端cd到 blog 目录下执行如下命令：

    git clone git@github.com:yscoder/hexo-theme-indigo.git themes/indigo  <下载主题> 
npm install hexo-renderer-less --save  <下载依赖>
npm install hexo-generator-feed --save  <下载依赖>

    修改hexo/_config.yml

    # 主题
theme: indigo

# 插件依赖
plugins: -hexo-renderer-less -hexo-generator-feed

    嘿嘿，主题到这里就安装成功了，然后就进入blog/themes/indigo目录下，打开_config.yml文件，开始配置博客中属于自己的内容。

    修改完之后，终端cd到blog目录下执行如下命令(每次部署了主题或文章的步骤)：

    hexo clean           //清除缓存文件 (db.json) 和已生成的静态文件 (public)
hexo g             //生成缓存和静态文件
hexo d             //重新部署到服务器

    然后在浏览器中打开网址http://wenghaobin.github.io（将wenghaobin换成你的用户名）能看到刚刚安装的主题和刚刚修改的内容。

    哈哈，到这里用hexo+github搭建的博客就基本搞定啦，跟着教程走，博客马上有！作为爱折腾的程序猿，怎么甘心到这里就结束呢？还要换一个属于自己的域名才对得起自己嘛，
    继续往下走。
