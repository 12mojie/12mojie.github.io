design my AndroidStudio

2016-05-12
  • tips
分享到：
  • 豆

从刚开始接触Android，就是在AndroidStudio上（以下简称为AS）写demo、做开发。直到今天我都一直使用自带的白色主题，因为感觉黑色的主题很伤眼，看久了眼睛很累，后来也只是找了个主题换了字体颜色，背景还是白色。今天傍晚突发奇想就上去Color Themes 找个黑色的代码高亮的主题，因为白色的主题代码的类型区分的不是很清楚，就想着有没有主题可以把代码里面的不同类型都看的清清楚楚，这样敲代码的时候思路也可以清晰一点（还可以装b 嘿嘿 >_<），最后看到Monokai Sublime Text 3 这一主题，很酷，代码高亮，区别度也很大，哈哈，就觉得是你了，立马下载。

下载完，打开AS：

File ——> Import Settings ——> 主题jar包所在位置 ——> 主题jar包 ——> restart AS

重新打开AS之后就看到下面这样的场景：

代码高亮，黑色冷酷，区分清晰，啧啧啧，赞一个。

哎！等等，左边和上边的导航栏怎么是白色的，原来这个主题只是改变字体颜色，要改变整个AS的主题才会全黑，嘿嘿，迫不及待动手了。

Mac：

Android Studio ——> Preference ——> Appearance & Behavior ——> Appearance ——> Themes ——> Darcula

Win:

Setting ——> IDE setting ——> Editor ——> Appearance ——> Themes ——> Darcula

恩，看看效果~

哎，怎么一改主题，我要的字体颜色怎么没了！这不科学，翻到字体的设置，手动设置字体颜色，哈哈，搞定，酷吧！

哎！等等，还有问题，为什么注释和静态方法都是灰色的，黑色背景的很难看清啊，不行，强迫症发作，我要改主题的颜色配置！解压主题jar包，拿到2个文件夹和1个执行文件，打开colors文件夹发现只有1个文件，看来就是你了。

打开一看，全是颜色的配置啊，眼花缭乱 @_@ ,怎么办？想想办法，有了，嘿嘿，先找到灰色颜色的配置是——75715E（在这里，我用的是mac自带的取色器，在灰色注释那里直接取的颜色再去找，这个我就不细说了，网上很多不同的取色器可以用），再全局搜索，一个个看标题写的是哪个的颜色，找到注释和静态办法的英文就用自己找到的喜欢的颜色改掉他的颜色（我下面只是一个例子，要找到说明是注释和静态办法的英文单词才好改哇）。

接下来就是要把自己改好的文件打包成jar文件导入AS就可以啦！这里简单讲解一下文件夹打包成jar包的方法（以mysimple文件为例）：

要打包mysimple文件夹，则先切换到此目录下，在终端中输入

jar cvf mysimple.war  *

则会将mysimple文件夹所有文件打包为一个名为mysimple.war的文件，就可以导入AS啦！

然而这里，有坑！

打包的时候，我把原本的jar包命名改了（避免篡改行为），导入AS就出错了，后来把所有的命名都改了，重新打包导入才成功！记得是所有的有关原本jar包的命名都要改！！！

最终的效果是这样的：

哈哈，帅爆了！我喜欢，给自己赞一个！
